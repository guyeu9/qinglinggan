# AI 模型管理页重构计划

## Context
当前AI设置详情页只能维护单个AI服务商配置，且模型列表是固定的默认值。用户需要：
1. 支持维护多个AI服务商配置
2. 从云端动态获取模型列表
3. 参考提供的设计图进行UI重构，但使用绿色UI规范

## Scope
涉及文件：`d:\trae\qinglinggan\前端原型图\8侧边栏-ai管理详情页\code.html`

## Current vs Target Analysis

### 当前问题
- 单表单结构，只能配置一个AI
- 模型选择是固定的下拉选项（gpt-4o, claude-3-5等）
- 没有服务商列表展示
- 没有添加/编辑/删除多个服务商的能力

### 目标设计（基于参考图）
参考图展示了一个完整的AI模型管理系统：
1. **顶部标签切换**：对话服务 / Embedding服务
2. **服务商列表**：显示多个已配置的服务商卡片
   - 单选按钮选择默认服务商
   - 显示服务商名称、模型名称、API Key（脱敏）
   - 编辑和删除按钮
3. **添加按钮**：右下角浮动按钮"添加服务商"
4. **添加/编辑弹窗**：
   - 快速选择模板（DeepSeek、通义千问、Claude等）
   - 服务商名称输入
   - API类型选择（OpenAI兼容等）
   - Base URL输入
   - API Key输入（带显示/隐藏切换）
   - 模型名称输入 + 加载按钮（从云端获取）
   - 保存按钮

### 交互逻辑
1. **页面加载**：显示已保存的服务商列表，默认选中第一个
2. **切换标签**：在"对话服务"和"Embedding服务"之间切换
3. **选择默认**：点击单选按钮设置默认服务商
4. **添加服务商**：
   - 点击"添加服务商"按钮打开弹窗
   - 可选择快速模板自动填充部分字段
   - 填写表单后点击保存
   - 新服务商添加到列表
5. **编辑服务商**：
   - 点击编辑按钮打开弹窗
   - 表单预填充当前数据
   - 修改后保存更新
6. **删除服务商**：
   - 点击删除按钮
   - 确认后从列表移除
7. **加载模型**：
   - 输入Base URL和API Key后
   - 点击"加载"按钮从云端获取可用模型列表
   - 模型名称变为下拉选择或自动填充

## UI 规范（绿色主题）

### 颜色系统
- 主色：`#6EE7B7` (primary)
- 深色：`#065F46` (primary-dark)
- 背景：`#F0FDF4` (background-light)
- 白色：卡片背景
- 边框：`border-primary/20`

### 样式规则
- 卡片：白色背景，圆角xl，无阴影或轻微阴影
- 按钮：主色背景，深色文字
- 输入框：白色背景，主色边框聚焦
- 标签：主色浅色背景，深色文字
- 选中状态：主色边框高亮

## Risks
1. **数据持久化**：当前是静态HTML，需要LocalStorage模拟数据存储
2. **云端API调用**：实际项目中需要后端支持，原型中使用模拟数据
3. **弹窗交互**：需要实现遮罩层、关闭逻辑、表单验证
4. **模型加载**：需要模拟异步加载过程
