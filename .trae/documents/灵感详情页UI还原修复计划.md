# 轻灵感详情页 UI 还原修复计划

## 一、问题概述

通过对比原型图截图与现有代码实现，发现存在多处严重差异。本计划将逐一拆解每个UI元素，确保100%还原设计。

---

## 二、逐元素对比分析

### 1. 顶部导航栏 (Top Navigation Bar)

| 元素 | 原型图 | 当前代码 | 差异 |
|------|--------|----------|------|
| **整体背景** | 纯白色，无模糊效果 | 使用了 `backdrop-blur-md` 毛玻璃效果 + 半透明背景 | ❌ 多了毛玻璃效果 |
| **左侧返回按钮** | 箭头返回图标 | 使用了 `arrow_back` | ✅ 正确 |
| **中间标题** | "灵感详情"，字体较细 | 使用了 `font-bold` | ❌ 字重太粗 |
| **右侧图标** | 分享图标 + 更多图标 | 正确 | ✅ 正确 |
| **分割线** | 无底部边框 | 有 `border-b` 底部边框 | ❌ 多了边框 |
| **布局** | 三栏布局 | 使用了 `max-w-2xl mx-auto` 限制宽度 | ❌ 不应该限制宽度 |

**修复方案：**
- 移除 `backdrop-blur-md` 和半透明背景
- 标题改为 `font-medium`
- 移除底部边框
- 移除 `max-w-2xl` 限制，使用全宽布局

## 四、代码结构优化

当前代码主要问题：
1. 使用了 `max-w-2xl mx-auto` 限制内容宽度，原型图是移动端全宽设计
2. 多处使用了蓝色系 (`accent`) 而非绿色系 (`secondary`)
3. 多处多了不必要的边框
4. 导航栏有毛玻璃效果

修复顺序：
1. 先修复整体布局（移除宽度限制）
2. 修复顶部导航栏
3. 修复AI解析区域颜色
4. 修复智能优化建议区域颜色
5. 移除所有多余的边框

修复完成后，需要：
1. 在浏览器中打开页面（移动端尺寸）

3. 检查每个元素的间距、颜色、圆角是否一致
4. 确保没有多余的边框、阴影或效果
