# 数据保存问题全面排查报告与修复计划

## Context
用户反馈：当新建多条记录时，系统仅保存并显示第一条记录，后续创建的记录既未在界面显示，也未正确存储到数据库中。

1. 数据保存逻辑
2. 数据库操作
3. 前端状态管理
4. API接口
5. 历史更新记录
6. 错误处理机制

```dart
Future<bool> saveIdea(String content, {List<String>? imagePaths}) async {
  // 1. 验证输入
  if (content.trim().isEmpty && (imagePaths == null || imagePaths.isEmpty)) {
    return false;
  }
  
  // 2. 创建新IdeaEntity (id=0, 表示新记录)

  final savedIdea = await ideaRepo.save(idea);
  
  // 4. 重新加载列表
  await loadIdeas();

  unawaited(taskQueue.enqueue(savedIdea.id));
  
  return true;
}
```

**结论**: 保存逻辑正确，每次都创建新的IdeaEntity (id=0)

```dart
Future<void> loadIdeas() async {

}
```

## 二、发现的问题

### 🔴 问题1: 轮询逻辑只支持单个任务 (根本原因)

**位置**: `lib/application/providers/home_provider.dart` 第236行

**问题描述**:
```dart
_pollingTimer?.cancel();  // 取消之前的轮询！
_pollingTimer = Timer.periodic(...);
```

当用户快速保存多条灵感时：
1. 保存第一条 -> 开始轮询第一条
2. 保存第二条 -> **取消第一条的轮询**，开始轮询第二条
3. 保存第三条 -> **取消第二条的轮询**，开始轮询第三条

**结果**: 只有最后一条灵感会被正确轮询到结果！之前的轮询被取消了！

---

### 问题2: AI分析更新可能覆盖用户修改 (中等)

**位置**: `lib/application/task_queue/ai_task_queue.dart` 第155-222行

**问题描述**:
当AI分析完成后，会更新Idea的categoryId和tagIds。如果用户在AI分析完成前修改了内容，AI更新可能会覆盖用户的修改。

**位置**: `lib/application/task_queue/ai_task_queue.dart` 第278-291行

**已确认**: 代码中已正确设置 `status: AnalysisStatus.completed`

## 三、根本原因分析

### 核心问题：轮询Timer被覆盖

**问题复现步骤**:
1. 用户保存灵感A
2. `saveIdea` 调用 `_pollAnalysisResultSilent(A.id)`
3. `_pollingTimer?.cancel()` 执行（无效果，因为是第一次）
4. `_pollingTimer` 开始轮询A的分析结果
5. 用户快速保存灵感B
6. `saveIdea` 调用 `_pollAnalysisResultSilent(B.id)`
7. `_pollingTimer?.cancel()` 执行 **取消A的轮询！**
8. `_pollingTimer` 开始轮询B的分析结果
9. 灵感A的AI分析完成，但轮询已被取消，无法检测到
10. 列表刷新时，灵感A没有标签和分类数据

---

## 四、修复方案

### 修复1: 支持多个并发轮询 (核心修复)

**文件**: `lib/application/providers/home_provider.dart`

将单个 `_pollingTimer` 改为 `Map<int, Timer>` 存储多个轮询：

```dart
class HomeNotifier extends StateNotifier<HomeState> {
  final Ref _ref;
  final Map<int, Timer> _pollingTimers = {};  // 改为Map存储
  
  @override
  void dispose() {
    for (final timer in _pollingTimers.values) {
      timer.cancel();
    }
    super.dispose();
  }
  
  void _pollAnalysisResultSilent(int ideaId) {
    // 取消该ideaId的旧轮询（如果存在）
    _pollingTimers[ideaId]?.cancel();
    
    _pollingTimers[ideaId] = Timer.periodic(const Duration(seconds: 2), (timer) async {
      // ... 轮询逻辑 ...
      
      if (analysis.status == AnalysisStatus.completed) {
        timer.cancel();
        _pollingTimers.remove(ideaId);
        await loadIdeas();  // 刷新列表
      }
    });
  }
}
```

### 修复2: AI更新时重新获取最新数据

**文件**: `lib/application/task_queue/ai_task_queue.dart`

```dart
// 更新前重新获取最新数据
final latestIdea = await _ideaRepository.getById(ideaId);
if (latestIdea != null) {
  final updatedIdea = latestIdea.copyWith(
    categoryId: categoryId,
    tagIds: tagIds,
  );
  await _ideaRepository.update(updatedIdea);
}
```
