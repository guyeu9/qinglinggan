1. 项目概述
1.1 项目名称
《轻灵感》灵感记录软件
1.2 项目背景
用户在日常生活、工作策划、摄影创作等场景中，常常会产生即时灵感，但这些灵感往往碎片化、分散化，容易遗忘，也难以在后续被高效复用。传统笔记工具虽然能够完成记录，但通常需要用户自行分类、整理和回顾，增加了认知负担，导致“记得下、用不上”。
本项目旨在打造一款以极简输入为核心、以AI自动整理与关联分析为能力支撑的个人灵感管理工具，帮助用户低成本记录、高效率沉淀、高价值复用灵感内容。
1.3 产品定位
一款极简快速记录 + AI自动整理 + 智能灵感关联的个人灵感笔记工具。
用户只需输入文字，系统自动完成分类、关联、分析，帮助用户构建可复用的灵感资产。
1.4 产品目标
本产品第一阶段目标如下：
1.	 降低用户记录灵感的操作门槛
2.	 确保灵感内容可被稳定保存，不丢失
3.	 基于AI完成自动分类、标签提取、历史关联分析
4.	 提供统一的AI分析入口，承载搜索、回顾、对话与整理能力
5.	 形成清晰、极简、低干扰的产品结构

2. 产品原则
2.1 记录优先：保存动作必须极速、稳定，不被AI分析流程阻塞。
2.2 操作极简：用户无需进行手动分类、手动打标签、手动整理。
2.3 AI能力统一承载：所有AI搜索、回顾、分析、对话能力统一归入“AI分析界面”，避免产品入口分散。
2.4 数据永久保存：所有记录必须可靠落库，支持完整导出和导入。增加回收站界面用于恢复和清空回收站
2.5 不打扰用户思路：尽量避免复杂分类、过度推荐、频繁提示，以免影响用户原始灵感记录过程。

3. 用户与使用场景
核心使用场景
1.	 用户打开应用后立即输入一条新灵感并保存  
2.	 用户保存后查看AI自动生成的分类和关联结果  
3.	 用户进入某条灵感详情页查看相关灵感和AI分析  
4.	 用户在AI分析界面中进行自然语言搜索  
5.	 用户在AI分析界面中查看近半个月的灵感回顾与总结  
6.	 用户导出全部历史记录用于备份或迁移

4. 产品范围
4.1 本期范围
本期版本包含以下核心能力：
•	极速文字记录
•	自动分类（固定3大类）
•	后台标签生成与关联展示
•	AI灵感关联分析
•	统一AI分析界面
•	时间流管理
•	灵感详情页
•	编辑、删除
•	数据导出与导入
•	回收站的恢复和清空
6. 详细功能需求

6.1 首页记录页
参考内容：
 
6.1.1 功能目标
提供最低操作成本的灵感输入入口，支持用户快速记录并立即保存。
6.1.2 页面结构
•	顶部：加号按钮
•	中部：时间流列表
•	底部：一行输入框 + 保存按钮
6.1.3 功能说明
1.	 用户打开应用后默认进入首页记录页  
2.	 底部输入框支持直接输入文字  
3.	 输入框随内容增多自动扩展高度  
4.	 点击保存按钮后立即完成内容写入数据库  
5.	 保存后自动清空输入框，回到待输入状态  
6.	 顶部加号按钮用于弹出详细输入卡片，用于补充更完整内容
6.1.4 交互要求
•	输入框始终处于可立即输入状态
•	保存操作不得被AI处理阻塞
6.1.5 保存后反馈
保存成功后，页面需自动展示一张轻量结果卡，内容包括：
•	自动归类结果
•	关联灵感数量
•	一句简短AI提示语
示例：
•	已归类：摄影爱好类
•	发现 2 条相关灵感
•	AI提示：与你此前记录的“雨夜街头拍摄”相关

6.2 自动分类
6.2.2 分类规则
系统仅提供以下固定三大类：
1.	 社交 / 旅行 / 惊喜类  
2.	 工作 / 创意策划类  
3.	 摄影爱好类
6.2.3 展示规则
•	前台界面仅展示三大类
•	用户不可自定义分类体系
•	每条灵感仅归属于一个主分类
•	暂不处理跨类、多类归属逻辑
6.2.4 后台标签策略
系统在后台自动生成标签，用于：
•	AI检索
•	关联分类展示
•	详情页展示
•	AI分析界面理解和检索
标签仅作为辅助理解和展示手段，不要求用户维护。

6.3 AI智能关联
6.3.1 功能目标
将新增灵感与历史灵感建立联系，提高灵感复用率和价值沉淀能力。
6.3.2 关联类型
AI关联统一由一个分析模块输出，包含以下三类关系：
1）相似关联
识别内容语义上接近、主题相似的灵感内容。
2）互补关联
识别内容可组合、可互相补充形成更完整方案的灵感内容。
3）演化/升级关联
识别新灵感是否属于历史灵感的延展、细化、升级版本。
6.3.3 输出内容
每次AI关联分析需输出以下内容：
•	关联灵感列表
•	每条关联的关系类型
•	关联原因说明
•	共同点
•	差异点
•	综合优化版本
6.3.4 展示位置
•	保存后轻量结果卡
•	灵感详情页
•	关联内容界面
•	AI分析界面

6.4 AI分析界面
6.4.1 功能目标
作为全局统一的AI能力入口，承载搜索、回顾、分析、对话能力，避免多功能分散。
6.4.2 功能范围
AI分析界面需要支持：
1.	 自然语言搜索  
2.	 历史灵感回顾  
3.	 相关灵感分析  
4.	 基于系统内全部数据的AI对话  
5.	 近半个月灵感内容总结  
6.	 指定时间范围内的灵感分析与提炼
6.4.3 使用说明
首页一个ai分析的按钮，点击后跳转新的界面，用户可以直接输入自然语言问题，例如：
•	我最近记录过哪些和旅行有关的灵感？
•	帮我分析最近半个月的摄影想法
•	找出与某条灵感相关的历史内容
•	这些灵感中哪些可以组合起来？
6.4.4 页面要求
•	支持对话形式输入问题
•	支持展示引用到的相关灵感列表
•	支持展示AI的总结、提炼、对比结果
•	支持点击跳转对应灵感详情页

6.5 时间流列表
6.5.1 功能目标
提供用户按时间顺序查看全部灵感的基础能力。
6.5.2 功能说明
•	按记录时间倒序排列
•	支持按三大分类筛选
•	每条记录展示完整信息。仅展示用户录入的内容
•	点击后进入灵感详情页
6.5.3 展示字段建议
•	灵感原文前若干字
•	记录时间
•	所属分类

6.6 灵感详情页
6.6.1 功能目标
为单条灵感提供完整的信息展示与AI解析结果查看入口。
6.6.2 页面结构
详情页分为以下四个层级：
1）原始记录区
展示用户最初保存的原始文本内容。
2）AI解析区
展示：
•	分类结果
•	自动标签
•	简单的分析总结
3）关联灵感列表区
展示与当前灵感相关的历史内容，按以下关系聚合：
•	相似
•	互补
•	演化/升级
4）AI分析输出区
展示：
•	共同点
•	差异点
•	综合优化版本
6.6.3 交互要求
•	页面保持信息清晰，不增加额外复杂按钮
•	允许从关联内容跳转到其他灵感详情页
•	保持阅读优先，而非过度交互

6.7 关联内容界面
6.7.1 功能目标
集中展示当前灵感的所有相关内容及其分组关系。
6.7.2 入口
用户可从灵感详情页底部进入关联内容界面。
6.7.3 功能说明
•	展示所有关联记录
•	按后台标签进行分组展示
•	支持查看某一标签下的相关灵感集合
6.7.4 示例分组
•	关联到摄影的几条
•	关联到美食的几条
•	关联到旅行的几条

6.8 编辑 / 删除
6.8.1 编辑
用户可以对已有灵感内容进行编辑。
编辑后需重新触发AI分析流程，并更新分类、标签、关联结果。
6.8.2 删除
用户可以删除某条灵感。
删除后需同步更新相关联记录的引用状态，避免出现无效关联项。

6.9 数据导入 / 导出
6.9.1 目标
保证用户数据可迁移、可备份、可控。
6.9.2 导出格式
支持以下两种导出方式：
1.	 Excel格式  
2.	 数据库原始格式
6.9.3 导入能力
支持导入上述格式的数据，并进入系统统一存储和分析流程。
6.9.4 要求
•	导出必须完整
•	导出数据与系统内数据一致
•	导入后可重新触发AI分析或使用已有结构化数据

7. AI技术方案设计

7.1 AI能力总览
本产品的AI能力主要由四个模块组成：
1.	 内容理解模块  
2.	 向量检索模块  
3.	 关系判断模块  
4.	 生成输出模块

7.2 模块设计
7.2.1 模块A：内容理解
职责：
•	自动分类
•	摘要提炼
•	标签生成
输入：
•	用户原始灵感文本
输出：
•	category
•	summary
•	tags
7.2.2 模块B：向量检索
职责：
•	为每条灵感生成embedding
•	将新灵感与历史灵感进行语义相似检索
输入：
•	当前灵感文本或摘要
•	历史灵感向量库
输出：
•	TopN相关灵感候选集
7.2.3 模块C：关系判断
职责：
对候选集进一步判断关系类型：
•	相似
•	互补
•	演化/升级
输入：
•	当前灵感
•	检索得到的候选灵感
•	标签、分类、摘要等结构化信息
输出：
•	关系类型
•	判断原因
•	置信度
7.2.4 模块D：生成输出
职责：
基于当前灵感与关联灵感，输出可展示给用户的分析结果，包括：
•	共同点
•	差异点
•	综合优化版本
输入：
•	当前灵感
•	关联灵感
•	关系类型判断结果
输出：
•	analysis.common
•	analysis.diff
•	analysis.merged_idea

7.3 AI输出格式规范
为了保证前端展示稳定和可控，AI输出必须采用固定结构化格式。
标准输出示例
{
  "category": "摄影爱好类",
  "summary": "夜景人像拍摄构图想法",
  "tags": ["夜景", "人像", "构图"],
  "related": [
    {
      "id": "note_001",
      "type": "相似",
      "reason": "两条内容都围绕夜景人像场景与氛围表达"
    },
    {
      "id": "note_002",
      "type": "互补",
      "reason": "该历史灵感可补充拍摄路线与时间安排"
    }
  ],
  "analysis": {
    "common": [
      "都关注夜景氛围表达",

    ],

  },
  "confidence": 0.86
}
  

8. AI准确性与稳定性设计
8.1 降低误判策略
为避免AI输出不准确，采用以下组合策略：
1.	向量召回 + 规则筛选
先通过embedding召回候选，再用规则缩小范围
2.	仅输出高相关TopN结果
避免展示过多低质量关联
3.	低置信度降级展示
对置信度较低的结果，仅显示“可能相关”，不做强解释
4.	结构化输出约束
限制AI自由发挥，确保展示一致性
8.2 可读性控制
•	统一字段名
•	固定前端渲染模板
•	统一摘要长度和分析粒度
•	限制单次分析输出条数和文字长度
9. 性能要求与异步策略
•	保存必须优先于AI分析完成
•	AI结果允许异步返回
•	下次进入应用时可补充展示已完成分析结果

9.2 异步处理策略
9.2.1 核心原则
先保存，后分析；记录动作不受AI影响。
9.2.2 处理流程
1.	 用户点击保存
2.	 内容立即写库
3.	 返回保存成功状态
4.	 后台异步触发AI分析任务
5.	 分析完成后写回结果表
6.	 前端轮询或下次进入页面时读取结果
9.2.3 超时处理
当大模型调用超时或失败时：
•	先完成基础分类与标签
•	深度关联分析进入延迟任务队列
•	用户无操作或下次打开时继续处理未完成任务

9.3 加载与缓冲体验
前端不展示AI分析状态，仅点击内容打开详情页后如果没有分析完成，也可以点击重新分析按钮进行分析。

10. （回收站）数据设计与安全要求
10.1 数据回收站
•	删除后原始内容进入回收站，避免误删，支持恢复（新增一个回收站的界面用于恢复）
10.2 数据导入导出要求
支持完整导出：
•	Excel格式
•	数据库原始格式
支持导入的数据库
10.4 数据一致性要求
•	原始内容、分类、标签、关联结果需保持一致性
•	删除、编辑操作需同步更新相关索引与关联关系

11. 研发实现建议
11.1 推荐技术实现流程
1.	 用户保存内容  
2.	 服务端立即写入主表  
3.	 写入成功后异步触发AI任务  
4.	 AI任务执行以下步骤：
￮	分类
￮	摘要
￮	标签生成
￮	embedding生成与入库
￮	相似检索
￮	关系判定
￮	分析结果生成
5.	 将AI结果写回分析表/关联表
6.	 前端读取并渲染结果卡、详情页、AI分析界面

11.2 推荐数据表拆分
可按以下逻辑拆分：
•	灵感主表
•	AI分析结果表
•	标签表
•	灵感标签关系表
•	向量索引表
•	灵感关联关系表
•	AI任务状态表
13. 风险点与注意事项
13.1 AI误判风险
风险：分类错误、关联不准、总结不稳定
应对：结构化输出 + 规则筛选 + 置信度控制
13.2 数据风险
风险：记录丢失、导出不完整、编辑后关联失效
应对：强制写库成功后才提示保存完成；建立任务补偿机制；维护关联重建逻辑

14. 版本边界说明
14.1 当前版本聚焦
当前版本只解决三个核心问题：
1.	 快速记录  
2.	 自动整理  
3.	 AI分析与关联
14.2 当前版本不扩展
不做复杂产品化延伸，不扩展为大型知识管理或多端协同系统。
