### 5.3 MVP 阶段 AI 流程（简化版）

```dart
// lib/application/ai/ai_analysis_service.dart
class AIAnalysisService {
  final AIUnderstandingService _understanding;
  final AIEmbeddingService _embedding;
  final IdeaRepository _ideaRepo;
  final AIAnalysisRepository _analysisRepo;
  
  /// MVP 阶段：仅做内容理解 + Embedding
  Future<void> runBasicAnalysis(int ideaId) async {
    final idea = await _ideaRepo.getById(ideaId);
    if (idea == null) return;
    
    try {
      // 1. 内容理解
      final understanding = await _understanding.analyze(idea.content);
      
      // 2. 生成 Embedding
      final embedding = await _embedding.generateEmbedding(idea.content);
      
      // 3. 保存结果
      await _ideaRepo.updateEmbedding(ideaId, embedding);
      await _ideaRepo.updateCategory(ideaId, understanding.categoryId);
      await _ideaRepo.updateTags(ideaId, understanding.tagIds);
      
      await _analysisRepo.save(AIAnalysis(
        ideaId: ideaId,
        categoryResult: understanding.category,
        tagsResult: understanding.tags,
        summary: understanding.summary,
        aiHint: understanding.aiHint,
        status: AnalysisStatus.completed,
      ));
      await _ideaRepo.updateAIStatus(ideaId, AIStatus.completed);
    } catch (e) {
      await _ideaRepo.updateAIStatus(ideaId, AIStatus.failed);
      rethrow;
    }
  }
}
```

## 六、本地任务队列系统

### 6.1 任务队列设计

```dart
// lib/application/task_queue/ai_task_queue.dart
class AITaskQueue {
  final Queue<AITaskModel> _queue = Queue();
  final AIAnalysisService _aiService;
  final AITaskRepository _taskRepo;
  bool _isProcessing = false;
  
  /// 入队
  Future<void> enqueue(int ideaId) async {
    final task = AITaskModel()
      ..ideaId = ideaId
      ..taskType = TaskType.basicAnalysis
      ..status = TaskStatus.pending;
    
    await _taskRepo.save(task);
    _queue.add(task);
    _processNext();
  }
  
  /// 处理队列
  Future<void> _processNext() async {
    if (_isProcessing || _queue.isEmpty) return;
    
    _isProcessing = true;
    final task = _queue.removeFirst();
    
    try {
      await _taskRepo.updateStatus(task.id, TaskStatus.processing);
      await _aiService.runBasicAnalysis(task.ideaId);
      await _taskRepo.updateStatus(task.id, TaskStatus.completed);
    } catch (e) {
      await _handleFailure(task);
    } finally {
      _isProcessing = false;
      _processNext();
    }
  }
  
  /// 失败处理
  Future<void> _handleFailure(AITaskModel task) async {
    task.retryCount++;
    
    if (task.retryCount < 3) {
      task.status = TaskStatus.pending;
      await _taskRepo.update(task);
      _queue.add(task);
    } else {
      task.status = TaskStatus.failed;
      await _taskRepo.update(task);
    }
  }
  
  /// 应用启动时恢复未完成任务
  Future<void> resumePendingTasks() async {
    final pendingTasks = await _taskRepo.getPendingTasks();
    _queue.addAll(pendingTasks);
    _processNext();
  }
}
```

```dart
// lib/application/usecases/create_idea.dart
class CreateIdeaUseCase {
  final IdeaRepository _ideaRepo;
  final AITaskQueue _taskQueue;
  
  Future<IdeaEntity> execute(String content) async {
    // 1. 立即保存
    final idea = IdeaEntity(
      id: 0,
      content: content,
      createdAt: DateTime.now(),

    final savedIdea = await _ideaRepo.save(idea);
```

## 九、关键技术决策说明

### 9.1 为什么 Domain 和 Data 分离？

**Domain（领域层）**
- 纯业务逻辑
- 不依赖任何框架
- 便于测试
- 便于未来更换数据库

**Data（数据层）**
- Isar 具体实现
- 包含 Schema 定义
- 实现 Repository 接口

### 9.2 为什么 AI 服务要拆分？

**单一职责原则**
- 每个服务只做一件事
- 便于测试
- 便于扩展
- 便于分阶段开发

### 9.3 为什么 MVP 只做内容理解 + Embedding？

**降低开发复杂度**
- 关系判断需要大量 Prompt 调优
- 综合生成需要复杂逻辑
- MVP 先验证核心价值
- 后续迭代再增强

这是一个**融合了 AI 能力完整性和 Flutter 架构规范性**的最终方案：
