# 回收站页面实现提案

## Context

根据原型图 4回收站，需要创建一个回收站页面，用于展示用户已删除的灵感记录。用户可以在回收站中查看已删除的内容、恢复灵感或彻底删除。

## Scope

### 涉及文件
1. **新建文件**:
   - `lib/presentation/pages/recycle_bin/recycle_bin_page.dart` - 回收站主页面

2. **可能涉及的现有文件**:
   - `lib/core/router/app_router.dart` - 需要添加回收站路由
   - `lib/core/router/route_names.dart` - 需要添加回收站路由名称

## Behavior

### 页面结构
1. **顶部导航栏**
   - 左侧：返回按钮（返回上一页）
   - 中间：标题"回收站"
   - 右侧：清空按钮（清空所有已删除内容）

2. **提示信息栏**
   - 显示提示信息："内容将在30天后自动彻底删除"
   - 使用柔和的背景色和图标

3. **已删除灵感卡片列表**
   - 每个卡片包含：
     - 顶部：标签（分类）和剩余天数
     - 中间：内容预览（最多显示4行）
     - 底部：删除时间
     - 操作按钮：彻底删除按钮、恢复按钮

### 视觉规范
- 使用 AppColors 颜色系统
- 支持暗黑模式
- 卡片样式与现有 InspirationCard 保持一致
- 剩余天数使用警告色（橙色）显示
- 彻底删除按钮使用错误色（红色）
- 恢复按钮使用主色（绿色）

### 数据结构
```dart
// 回收站项目数据模型
class RecycleBinItem {
  final String id;
  final String content;
  final String category;
  final DateTime deletedAt;
  final int daysRemaining; // 剩余天数
}
```

### 交互行为
1. **返回按钮**: 返回上一页
2. **清空按钮**: 显示确认对话框，确认后清空所有内容
3. **彻底删除按钮**: 显示确认对话框，确认后永久删除该灵感
4. **恢复按钮**: 将该灵感恢复到正常列表
5. **卡片点击**: 可查看灵感详情

## Risks

1. **数据风险**: 彻底删除操作不可逆，需要二次确认
2. **性能风险**: 如果回收站内容过多，需要考虑分页加载
3. **兼容风险**: 需要确保暗黑模式下的颜色对比度符合可访问性标准

## 依赖

- `AppColors` - 应用颜色系统
- `AppTheme` - 主题配置（间距、圆角等）
- `material_symbols_icons` - 图标库
- `go_router` - 路由导航
